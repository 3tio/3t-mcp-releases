Accepting the End User License Agreement is now required. The first CLI data command or MCP tool call opens a browser page with the EULA (v1.0) and waits for you to accept. The acceptance is recorded locally and reused afterwards, so it happens once per machine.

connections.yaml validation got stricter. Unknown keys are now rejected — at the top level and inside a connection entry — where they used to be ignored. If your file carries a stray or misspelled key, this release refuses to load it and names the key and line. This is deliberate: connections add and connections remove rewrite the file from what they parsed, so a misspelling such as passwrd: was silently dropped from the file on the next write.

Fixed
- connections add now works on a connections.yaml that exists but is empty. It used to fail with a missing field parse error, leaving you to write the connections: key by hand — the very key the command exists to write. Empty, whitespace-only and comment-only files are all accepted.
- The message shown when the config is missing or has no entries now covers the empty file as well, suggests the connections add command that fixes it, and quotes the path so it survives spaces.
- The update notice now appears on the first run after a version is released, instead of the run after that.

Added
- Clearer per-tool help text and connection error hints in the CLI.

Docs
- README has a new Per-project configuration section: giving each project its own connections.yaml via --config, pointing an MCP client registration at it, and passing credentials through the client's env block. It also states plainly that .env files are not read — env:VAR is resolved from the environment of the process running the binary.
